## Story
**As a** Anderson (gestor da 4G),
**I want** configurar os parâmetros de busca antes de iniciar uma prospecção,
**so that** eu possa encontrar empresas na região e quantidade que eu quiser.

## Acceptance Criteria
1. Formulário com campos: país, estado, cidade e quantidade (1 a 100)
2. Botão "Iniciar Busca" que aciona o fluxo n8n via webhook
3. Estado de loading exibido enquanto o Apify busca as empresas
4. Busca salva na tabela `searches` com status `PENDENTE` e data
5. Validação: todos os campos obrigatórios antes de enviar

## 🤖 CodeRabbit Integration
> **CodeRabbit Integration**: Disabled
>
> Validação de qualidade usará revisão manual com foco na integração com n8n e validação de formulário.

  - [ ] Criar `src/app/api/searches/route.ts`

### Validações Obrigatórias
[Source: docs/architecture.md#segurança]
- `quantidade`: inteiro entre 1 e 100 (CHECK constraint no banco)
- `pais`, `estado`, `cidade`: strings não-vazias
- Autenticação: rota protegida pelo middleware (Story 1.3)

### Testing
[Source: docs/architecture.md#testes]
- Testar formulário com todos os campos preenchidos → busca criada, n8n acionado
- Testar formulário com campo vazio → erro de validação exibido
- Testar quantidade = 0 ou > 100 → erro de validação exibido
- Testar falha do webhook n8n → mensagem de erro exibida ao Anderson
