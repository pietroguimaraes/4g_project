## Story
**As a** sistema,
**I want** que o fluxo principal n8n use o novo banco e o dicionário oficial de status,
**so that** prospecção e qualificação usem o dashboard como fonte da verdade — e o Google Sheets seja aposentado.

## Acceptance Criteria
1. Nós do Google Sheets substituídos por chamadas HTTP à API do dashboard (com API key `x-api-key`)
2. Status gravados conforme dicionário oficial: `PROSPECTADOS`, `INTERESSE`, `DESCARTADOS`
3. Fluxo testado end-to-end: busca → aprovação → WhatsApp → qualificação → Kanban atualizado
4. Google Sheets não é mais escrito ou lido por este fluxo

## 🤖 CodeRabbit Integration
> **CodeRabbit Integration**: Disabled
>
> Validação será teste end-to-end manual do fluxo n8n completo com as credenciais reais.

## Dev Notes

### URL Base do Dashboard
```bash
# Produção (Vercel)
DASHBOARD_URL=https://4g-project.vercel.app

# Desenvolvimento local (para testar com n8n local)
DASHBOARD_URL=http://localhost:3000
```

### Mapa de Substituições
[Source: docs/architecture.md#api-rest]
| Operação Antiga (Sheets) | Nova Chamada HTTP |
|--------------------------|-------------------|
| Gravar empresa encontrada | POST /api/leads |
| Ler empresas para disparar WhatsApp | GET /api/leads?status=PROSPECTAR |
| Marcar WhatsApp enviado | PATCH /api/leads/{telefone} → PROSPECTADOS |
| Gravar qualificação da IA | PATCH /api/leads/{telefone} → INTERESSE + categoria + nota |
| Marcar como descartado | PATCH /api/leads/{telefone} → DESCARTADOS |

### Configuração do Nó HTTP Request no n8n
```
Method: POST (ou PATCH ou GET)
URL: https://4g-project.vercel.app/api/leads
Headers:
  x-api-key: {{$env.N8N_API_KEY}}
  Content-Type: application/json
Body: (JSON conforme tabela acima)
```

### Variáveis de Ambiente no n8n
- `N8N_API_KEY`: mesmo valor configurado no Vercel — nunca hardcoded no fluxo
- Configurar em: n8n → Settings → Variables → criar `N8N_API_KEY`

### Dicionário de Status a Usar
[Source: docs/prd.md#dicionário-oficial-de-status]
```
Apify encontra empresa    → LOCALIZADOS  (POST /api/leads, sem body de status)
Anderson aprova           → PROSPECTAR   (via dashboard)
WhatsApp enviado          → PROSPECTADOS (PATCH pelo n8n)
IA qualifica com interesse → INTERESSE   (PATCH pelo n8n)
Sem fit / bot / opt-out   → DESCARTADOS  (PATCH pelo n8n)
```

### Testing
[Source: docs/architecture.md#testes]
- Testar POST /api/leads via n8n com API key válida → lead aparece no Painel de Aprovação
- Testar PATCH status → Kanban atualiza em tempo real
- Testar com API key inválida → n8n recebe 401, fluxo trata o erro
- Testar fluxo completo end-to-end com 1 lead real

- Story 2.3 concluída (endpoint PATCH /api/leads/[telefone] ativo)
- Story 3.1 concluída (Kanban exibindo leads em tempo real)
- `N8N_API_KEY` configurada tanto no Vercel quanto no n8n
