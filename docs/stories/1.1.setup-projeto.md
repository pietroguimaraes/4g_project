## Story
**As a** desenvolvedor,
**I want** o projeto Next.js configurado com Supabase e Vercel,
**so that** a base técnica esteja pronta para o desenvolvimento do dashboard 4g_project.

## Acceptance Criteria
1. Projeto Next.js criado com TypeScript e Tailwind CSS via `create-next-app`
2. Supabase conectado ao projeto (variáveis de ambiente configuradas no `.env.local`)
3. Deploy funcionando no Vercel com URL pública acessível
4. Repositório Git inicializado e conectado ao GitHub

## Tasks / Subtasks

- [ ] Task 1 — Criar projeto Next.js (AC: 1)

  - [ ] Criar cliente Supabase em `src/lib/supabase/client.ts` (browser)

  - [ ] Criar `src/lib/api/` (vazio por ora)
  - [ ] Criar `supabase/migrations/` (vazio por ora)
  - [ ] Criar `docs/stories/` (já existe)

- [ ] Task 4 — Configurar deploy na Vercel (AC: 3)
  - [ ] Criar conta/projeto na Vercel e conectar ao repositório GitHub
  - [ ] Configurar variáveis de ambiente na Vercel (mesmas do `.env.local`)

- [ ] Task 5 — Inicializar Git e GitHub (AC: 4)

  - [ ] Fazer primeiro commit e push

## Dev Notes

### Stack
[Source: docs/architecture.md]
- **Framework:** Next.js 14+ com App Router e TypeScript 5.x
- **Estilização:** Tailwind CSS 3.x
- **Componentes UI:** shadcn/ui (instalar na inicialização)
- **Banco de dados:** PostgreSQL via Supabase
- **Drag & Drop:** @dnd-kit/core + @dnd-kit/sortable
- **Estado:** Zustand 4.x
- **Deploy:** Vercel (gratuito)

### Estrutura de Pastas Alvo
