## Story
**As a** Anderson (gestor da 4G),
**I want** mover leads entre colunas do Kanban arrastando e marcar leads qualificados como transferidos,
**so that** eu possa gerenciar o funil manualmente e registrar o handoff quando assumo o lead como Closer.

## Acceptance Criteria
1. Cards podem ser arrastados entre colunas do Kanban
2. Status do lead atualizado no banco imediatamente após o drop
3. Cards na coluna `INTERESSE` exibem botão "Transferir para Closer"
4. Ao clicar em "Transferir para Closer", status muda para `TRANSFERIDOS`
5. Indicação visual clara durante o arraste (card semitransparente, coluna de destino destacada)

## 🤖 CodeRabbit Integration
> **CodeRabbit Integration**: Disabled
>
> Validação de qualidade usará revisão manual com foco na experiência de drag and drop e atualização imediata do banco.

## Dev Notes

### Biblioteca de Drag and Drop
[Source: docs/architecture.md#tech-stack]
- `@dnd-kit/core` + `@dnd-kit/sortable` (já instalados na Story 1.1)
- `DndContext`: contexto que engloba todo o Kanban
- `useDraggable(id)`: torna o KanbanCard arrastável
- `useDroppable(id)`: torna a KanbanColumn receptora de drops
- `onDragEnd({ active, over })`: captura `leadId` (active.id) e `targetStatus` (over.id)

### Optimistic Update
```typescript
// Mover card visualmente antes da resposta da API
// 1. Atualizar estado local imediatamente
setLeads(prev => moveLeadToColumn(prev, leadId, newStatus))
// 2. Chamar API em background
try {
  await updateLeadStatus(telefone, newStatus)
} catch {
  // 3. Reverter se falhar
  setLeads(prev => moveLeadToColumn(prev, leadId, originalStatus))
  toast.error('Não foi possível mover o lead')
}
```
