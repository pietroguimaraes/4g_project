## Story
**As a** Anderson (gestor da 4G),
**I want** adicionar manualmente uma empresa para prospecção informando nome e telefone,
**so that** eu possa incluir leads que não vieram da busca automática pelo Apify.

## Acceptance Criteria
1. Botão "Adicionar Empresa" visível no Painel de Aprovação
2. Formulário com dois campos: nome da empresa e telefone
3. Empresa salva com status `PROSPECTAR` diretamente (pula a etapa de aprovação)
4. WhatsApp de prospecção disparado imediatamente via n8n, passando para `PROSPECTADOS`
5. Empresa aparece no Kanban na coluna `PROSPECTADOS` após o disparo

## 🤖 CodeRabbit Integration
> **CodeRabbit Integration**: Disabled
>
> Validação de qualidade usará revisão manual com foco na validação de telefone e integração n8n.

### Reutilização do Endpoint POST /api/leads
[Source: Story 2.2 + docs/architecture.md#api-rest]
- O mesmo endpoint criado na Story 2.2 é reutilizado aqui
- Diferença: Anderson chama via sessão Supabase (não API key do n8n)
- O endpoint aceita `status` explícito no body — se não enviado, padrão é `LOCALIZADOS`
- Para adição manual: enviar `status: 'PROSPECTAR'` diretamente

```typescript
// Chamada do frontend (Anderson autenticado via sessão)
POST /api/leads
Cookie: session
{
  "empresa": "Loja do Anderson",
  "telefone": "5511999998888",
  "status": "PROSPECTAR"
}
```

### Fluxo após Adição Manual (AC: 4, 5)
[Source: docs/architecture.md#workflow-1]
```
Anderson adiciona empresa manualmente
  → POST /api/leads { status: 'PROSPECTAR' }
  → Lead salvo em PROSPECTAR no banco
  → n8n detecta lead em PROSPECTAR e dispara WhatsApp
  → n8n atualiza para PROSPECTADOS
  → Supabase Realtime → card aparece no Kanban em PROSPECTADOS
```
> **Nota:** A empresa adicionada manualmente NÃO aparece no Painel de Aprovação — já entra diretamente em `PROSPECTAR` e depois em `PROSPECTADOS` no Kanban.
