## Story
**As a** sistema,
**I want** que o fluxo de Reengajamento Mensal use o novo banco,
**so that** a aposentadoria do Google Sheets seja completa e todos os 3 fluxos n8n usem o dashboard como fonte da verdade.

## Acceptance Criteria
1. Nós do Google Sheets substituídos por chamadas HTTP à API do dashboard (com API key)
2. Lógica de elegibilidade mantida: máximo 3 reengajamentos, mínimo 30 dias desde o último contato
3. Leads elegíveis lidos do status `PROSPECTADOS` (inclui leads que já receberam reengajamentos anteriores)
4. Google Sheets não é mais usado por nenhum dos 3 fluxos n8n — aposentadoria completa

## 🤖 CodeRabbit Integration
> **CodeRabbit Integration**: Disabled
>
> Validação será teste manual do fluxo de reengajamento com leads reais no banco.

## Dev Notes

### Lógica de Elegibilidade para Reengajamento
[Source: fluxo de reengajamento existente]
```
Leads elegíveis para reengajamento mensal:
  - status = 'PROSPECTADOS' (ainda em acompanhamento)
  - qtd_reengajamentos < 3 (máximo 3 tentativas de reengajamento)
  - data_followup <= NOW() - 30 dias (pelo menos 30 dias desde o último contato)
  - Se data_followup for NULL: usar data_coleta como referência
```

### Campos do Banco Usados
[Source: docs/architecture.md#schema]
```sql
-- Campos relevantes em leads para o fluxo de reengajamento
status                TEXT       -- Deve ser 'PROSPECTADOS'
qtd_reengajamentos    SMALLINT   -- Contador (máx 3)
data_followup         TIMESTAMPTZ -- Data do último contato (follow-up ou reengajamento)
data_coleta           TIMESTAMPTZ -- Fallback se data_followup for null
```

### Mapa de Substituições
| Operação Antiga (Sheets) | Nova Chamada HTTP |
|--------------------------|-------------------|
| Ler leads elegíveis | GET /api/leads?status=PROSPECTADOS |
| Gravar reengajamento enviado | PATCH /api/leads/{telefone} → { qtd_reengajamentos: N+1, data_followup: now } |

### Incremento do Contador no n8n
```javascript
// Expressão no nó HTTP Request do n8n para incrementar o contador
// O valor atual de qtd_reengajamentos vem da resposta do GET /api/leads
Body: {
  "qtd_reengajamentos": {{ $json.qtd_reengajamentos + 1 }},
  "data_followup": "{{ $now }}"
}
```

### Checklist de Aposentadoria do Google Sheets (AC: 4)
Ao finalizar esta story, verificar:
- [ ] Fluxo Principal: zero nós Google Sheets ativos
- [ ] Fluxo Follow-up: zero nós Google Sheets ativos
- [ ] Fluxo Reengajamento: zero nós Google Sheets ativos
- [ ] Google Sheets não recebe nenhuma escrita em nenhum fluxo
- [ ] Todos os dados estão no Supabase

### Testing
[Source: docs/architecture.md#testes]
- Criar lead em PROSPECTADOS com `data_followup` = 31 dias atrás e `qtd_reengajamentos = 0` → deve receber reengajamento
- Criar lead com `qtd_reengajamentos = 3` → não deve receber reengajamento
- Confirmar que `qtd_reengajamentos` incrementa de 0 para 1 após envio
- Confirmar que `data_followup` é atualizada no banco após o envio
- Após todos os testes: confirmar que Google Sheets está sem nenhum dado novo
