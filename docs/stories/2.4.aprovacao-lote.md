## Story
**As a** Anderson (gestor da 4G),
**I want** aprovar todas as empresas pendentes de uma vez com um único botão,
**so that** eu possa agilizar quando confio nos resultados da busca e não quero revisar card por card.

## Acceptance Criteria
1. Botão "Aprovar Todos" visível no Painel de Aprovação (visível apenas quando houver leads `LOCALIZADOS`)
2. Diálogo de confirmação antes de executar: "Aprovar X empresas?"
3. Todas as empresas `LOCALIZADOS` movidas para `PROSPECTAR` em uma única operação
4. n8n dispara WhatsApp para todas, passando para `PROSPECTADOS`
5. Painel esvaziado após aprovação em lote (sem leads `LOCALIZADOS` restantes)

### Endpoint PATCH /api/leads/bulk-approve

```typescript
// PATCH /api/leads/bulk-approve
// Autenticação: sessão Supabase (Anderson)
// Body: {} (vazio — opera em todos os LOCALIZADOS)
// Response: { updated: 42 }
```

### Operação no Banco
```sql
-- Atualização em lote — uma única query
UPDATE leads
SET status = 'PROSPECTAR', updated_at = NOW()
WHERE status = 'LOCALIZADOS'
RETURNING id;
```

### Fluxo após Bulk Approve (AC: 4)
[Source: docs/architecture.md#workflow-1]
```
Anderson confirma "Aprovar Todos"
  → PATCH /api/leads/bulk-approve
  → Todos LOCALIZADOS → PROSPECTAR
  → n8n lê todos em PROSPECTAR e dispara WhatsApp para cada um
  → n8n atualiza cada um para PROSPECTADOS
  → Supabase Realtime notifica Kanban
```
> **Nota:** O n8n processa `PROSPECTAR` em lote por polling. O dashboard não chama o n8n diretamente nesta operação.
