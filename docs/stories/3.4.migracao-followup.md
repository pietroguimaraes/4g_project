## Story
**As a** sistema,
**I want** que o fluxo de Follow-up (D+3 e D+7) use o novo banco,
**so that** o acompanhamento automático de leads `PROSPECTADOS` sem resposta continue funcionando após a aposentadoria do Google Sheets.

## Acceptance Criteria
1. Nós do Google Sheets substituídos por chamadas HTTP à API do dashboard (com API key)
2. Fluxo lê leads com status `PROSPECTADOS` e filtra por prazo (D+3 e D+7 desde `data_coleta`)
3. Status e datas atualizados corretamente no banco após cada follow-up enviado
4. Google Sheets não é mais usado por este fluxo

## 🤖 CodeRabbit Integration
> **CodeRabbit Integration**: Disabled
>
> Validação será teste manual do fluxo de follow-up com leads reais no banco.

## Dev Notes

### Lógica de Filtragem D+3 / D+7
[Source: fluxo de follow-up existente]
```
Leads elegíveis para follow-up D+3:
  - status = 'PROSPECTADOS'
  - data_coleta <= NOW() - 3 dias
  - data_followup IS NULL (nunca recebeu follow-up)

Leads elegíveis para follow-up D+7:
  - status = 'PROSPECTADOS'
  - data_coleta <= NOW() - 7 dias
  - data_followup <= NOW() - 4 dias (recebeu D+3 há pelo menos 4 dias)
```
> **Nota:** A lógica exata de filtragem já existe no fluxo n8n atual — apenas substituir a fonte de dados (Sheets → API).

### Campos do Banco Usados
[Source: docs/architecture.md#schema]
```sql
-- Campos relevantes em leads para o fluxo de follow-up
data_coleta     TIMESTAMPTZ   -- Quando o Apify encontrou (base para D+3/D+7)
data_followup   TIMESTAMPTZ   -- Quando o último follow-up foi enviado
status          TEXT          -- Deve ser 'PROSPECTADOS' para ser elegível
```

### Mapa de Substituições
| Operação Antiga (Sheets) | Nova Chamada HTTP |
|--------------------------|-------------------|
| Ler leads para follow-up | GET /api/leads?status=PROSPECTADOS |
| Marcar follow-up D+3 enviado | PATCH /api/leads/{telefone} → { data_followup: now } |
| Marcar follow-up D+7 enviado | PATCH /api/leads/{telefone} → { data_followup: now } |

### Endpoint PATCH já Existente
- O PATCH `/api/leads/[telefone]` criado na Story 2.3 já suporta atualizar `data_followup`
- Verificar que o endpoint aceita o campo `data_followup` no body (adicionar se necessário)

### Testing
[Source: docs/architecture.md#testes]
- Criar lead em PROSPECTADOS com `data_coleta` = 4 dias atrás → deve receber follow-up D+3
- Criar lead em PROSPECTADOS com `data_coleta` = 8 dias atrás e `data_followup` = 5 dias atrás → deve receber follow-up D+7
- Lead em INTERESSE → não deve receber follow-up
- Confirmar que `data_followup` é gravada no banco após cada envio

### Pré-requisito
- Story 3.3 concluída (fluxo principal migrado, endpoints validados)
- Campo `data_followup` disponível na tabela `leads` (Story 1.2)
