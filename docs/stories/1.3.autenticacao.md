## Story
**As a** Anderson (gestor da 4G),
**I want** fazer login com email e senha no dashboard,
**so that** eu possa acessar o sistema com segurança de qualquer lugar.

## Acceptance Criteria
1. Tela de login com campos de email e senha
2. Login funcional via Supabase Auth — credenciais válidas redirecionam ao dashboard
3. Sessão mantida entre visitas — não precisa logar toda vez que abre o navegador
4. Rotas do dashboard bloqueadas para usuários não autenticados — redirecionam para `/login`
5. Botão de logout disponível no layout do dashboard

## 🤖 CodeRabbit Integration
> **CodeRabbit Integration**: Disabled
>
> Validação de qualidade usará revisão manual com foco em segurança de autenticação.

## Dev Notes

### Padrão de Autenticação
[Source: docs/architecture.md#segurança]
- Autenticação via **Supabase Auth** — email + senha
- Sessão mantida via cookies (não localStorage) para funcionar com Server Components
- Usar `@supabase/ssr` — pacote oficial para Next.js App Router
- NUNCA usar `@supabase/auth-helpers-nextjs` (deprecado)

### Estrutura de Arquivos
[Source: docs/architecture.md#estrutura-do-projeto]

### Cliente Supabase (Browser)
[Source: docs/architecture.md]
```typescript
// src/lib/supabase/client.ts
import { createBrowserClient } from '@supabase/ssr'

export function createClient() {
  return createBrowserClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  )
}
```

### Segurança
[Source: docs/architecture.md#segurança]
- Senha nunca armazenada no código — apenas no Supabase Auth
- `SUPABASE_SERVICE_ROLE_KEY` nunca exposta ao browser
- Sessão por cookie httpOnly (gerenciado pelo @supabase/ssr automaticamente)


### Testing
[Source: docs/architecture.md#testes]
- Testar login com credenciais corretas → redireciona para `/aprovacao`
- Testar login com credenciais erradas → exibe mensagem de erro
- Testar acesso a `/aprovacao` sem login → redireciona para `/login`
- Testar logout → redireciona para `/login` e bloqueia acesso ao dashboard
